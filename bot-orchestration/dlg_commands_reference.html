DLG commands reference

DLG is a declarative language used to define conversation flows. Following you'll find a summary of the available commands.

User interaction​

say​

Command used to send output to the user. Any context variable in the text to say will be replace by its value (see examples below).

Syntaxsay <text or key>
Parameters
PositionOptionalDescription
1false
Text or translation key matching the text to send to the user

Usage examples

say Hello world!
to just send the 'Hello world!' text to the user
say greeting
where greeting is a key defined in the your language file
say Hi {{userName}}
will replace {{userName}} by the value of evaluating the expression userName using the variables available in the context, if any, and send the resulting text to the user
say Hi {{user.name}}
will replace {{user.name}} by the value of evaluating the expression user.name using the variables available in the context.

ask​

Used to gather input from the user. When used, conversation flow will stop processing until some input is received from the user.

Syntaxask <variable name>
Parameters
PositionOptionalDescription
1true
Name of the variable where user input will be stored

Usage examples

ask
just get the input and let following commands in the flow use it
ask type_of_food
get the input from the user and store it in the variable type_of_food

nlp​

Sends de current user input to the NLP for evaluation. If an intent is matched, it's answer will be evaluated following these rules:

  • if the answer starts with '/', the bot will look for an existing dialog with the same name and execute it
  • otherwise, the answer will be send to the user the same way as it we had a say <answer>

By default, any intent in the corpus can be matched, however, the nlp command allows you to narrow the list of intents available in a user interaction. This way, if the user input matches a not allowed intent, it will be handled as if there is no matched intent.

multilanguage bots

When the user input is evaluated by the NLP, in case there is more than one language available, the NLU manager will try to guess the language of the input before the NLP evaluates it.

Syntaxnlp <allowed intents list>
Parameters
PositionOptionalDescription
1true
Space-separated list of allowed intents. Leave blank to allow any intent of the corpus to be matched

Usage examples

nlp
evaluate the user input using the NLP and behave according to the answer in the matching intent
nlp intent1 intent2
as the previous example, but only intents intent1 and intent2 can be matched

card​

Sends an adaptive card to the user.

Syntaxcard <card name>
Parameters
PositionOptionalDescription
1false
Name of the card to send

Usage examples

card initialChoose
sends the card with name initialChoose to the user

suggest​

Suggestions, also known as suggested actions, can be thought as "not so good looking cards", they are easier to build, though. As expected, suggestions are meant to suggest one or more possible responses or choices to the user, and are defined using a pipe-separate list. When placing a suggestion, it will be attached to the next message send to the user, and he will receive as many buttons as suggestions placed.

Syntaxsuggest <suggestion 1>|<suggestion 2>...
Parameters
PositionOptionalDescription
1false
pipe-separated list of suggestions

Usage examples

suggest Option 1|Option 2
Next message send to the user will include two buttons "Option 1" and "Option 2". When one of them is clicked, the buttons text will be sent to the bot as the user input
info

Directline connector and Microsoft Bot Framework connectors both support suggestions. You can check Microsoft Bot Framework documentation to get more info about suggested actions

Flow management​

dialog​

A dialog is a group of commands defined in a the flow of the conversation. The goal of having dialogs is to be able to execute sets of commands without the need of copy/pasting the command sequence everywhere where is needed. We can think of them as if tey where functions

Syntaxrun <dialog_name>
Parameters
PositionOptionalDescription
1false
name of the dialog to execute

Usage examples

dialog place_order
defines the start of the place_order dialog definition. All commands below will be part of the place_order dialog until another dialog command is found, or the end the file is reached
tip

In general, dialogs are used to track conversation paths. So, if you are integrating your bot with the AXA Bots application, or plan to get some analytics like which paths of your conversation are more used, or where the users tend to stop using the bot, try also to organize them in a smart way by making them meaningful from a business point of view.

run​

run command will start executing another dialog, once that dialog is finished, the bot will continue where it left

Syntaxrun <dialog_name>
Parameters
PositionOptionalDescription
1false
name of the dialog to execute

Usage examples

run placeOrder
would run the placeOrder dialog
run placeOrder
say Do you want to place another order?
would run the placeOrder dialog, and, once that dialog has finished its execution, would ask the user if he wants to place another order

restart​

Restarts the conversation and clean up the dialog stack

conditional execution​

It is possible to define commands, in the conversation flow, to be executed only under certain conditions. In order to do that we can use the following syntax

[<condition>] <command>

The condition to be evaluated will take into account any variable available in the context. For instance, if we've requested the user to tell the bot his age, we could do something like this:

[age >= 18] say You can drive
[age <18] say You still have to wait {{ 18 - age }} years to drive!
tip

Unfortunately, in the current version multiline conditions are not supported, so, if you need a piece of the conversation to be conditionally executed, you may create a dialog and do something like

[<condition>] run <dialog_name>

Flow customization​

call​

Use the call command to execute a custom action at any point of a conversation

Syntaxcall <custom action name>
Parameters
PositionOptionalDescription
1false
name of the custom action to execute

Usage examples

call createCase
executes the createCase custom action

set​

The set command allows you to define a variable within the context. Remember that context variables are available all over the flow, and can be referenced in many places conditions, say commands, custom actions, custom validators, etc.

Syntaxset <var name> <value>
Parameters
PositionOptionalDescription
1false
name of the variable to set
2false
value to set

Usage examples

set count 5
set count variable to 5
set user {name: 'John', surname: 'Doe'}
say {{user.name}}
set user variable to have the JSON value {name: 'John', surname: 'Doe'} and says 'John'
set numbers []
say numbers: {{ numbers.length }}
set numbers variable with an empty array and says 'numbers length: 0'
set numbers [1,2,3]
say numbers: {{ numbers.length }}
set numbers variable to have an array with values and says 'numbers length: 3'

inc​

Given a context variable, it will be increased by the specified amount. If no amount is specified, it will be increased by 1. If var does not exist, it will be created and set to 1.

Syntaxinc <var name>
Parameters
PositionOptionalDescription
1false
name of the variable to increment
2true
amount to increment (defaults to 1)

Usage examples

inc myVar
myVar value before the command myVar value after the command
1213
'hello''hello1'
undefined1
 
inc myVar 5
myVar value before the command myVar value after the command
1217
'hello''hello5'
undefined5

dec​

Given a context variable, it will be decreased by the specified amount. If no amount is specified, it will be decreased by 1. If var does not exist, it will be created and set to -1.

Syntaxdec <var name>
Parameters
PositionOptionalDescription
1false
name of the variable to decrease
2true
amount to decrease (defaults to -1)

Usage examples

dec myVar
myVar value before the command myVar value after the command
1211
'hello'NaN
undefined-1
 
dec myVar 5
myVar value before the command myVar value after the command
127
'hello'NaN
undefined-5
```
